Missão · 01 — O Desafio DevOps

Do Localhost para a Nuvem.

O maior gargalo de desenvolvimento é a transição entre a máquina do desenvolvedor e o ambiente de produção. Nossa missão em DevOps foi criar uma infraestrutura replicável, segura e altamente disponível para hospedar a inteligência do AstroFarm na nuvem.

  • ↳Paridade de Ambiente. O uso de contêineres elimina falhas de dependência no servidor.
  • ↳Segurança. Credenciais (como a senha do Oracle) isoladas em variáveis de ambiente (.env), fora do código.
  • ↳Governança na Nuvem. Regras restritas de Firewall (NSG) na Azure, liberando apenas as portas vitais (22, 1521, 8080).

A Rede na Azure.

A segurança da rede baseia-se no bloqueio por padrão. Apenas os serviços essenciais foram mapeados nas portas do host: a API .NET escutando na porta 8080 e o Oracle na porta 1521.

Missão · 03 — Automação

Contêineres & Infraestrutura.

Docker Engine

Empacotamos a aplicação C# em uma imagem isolada. O contêiner roda sempre da mesma forma, sem depender das bibliotecas do servidor host.

Docker Compose

Orquestração declarativa. Subimos o banco Oracle e a API C# simultaneamente com um único comando, garantindo a ordem de inicialização.

Shell Scripting

Para agilizar o deploy na VM Linux, criamos um script de automação (`.sh`) que faz o clone do repositório, constrói a imagem e sobe a stack.

Gestão de Segredos

Uso de arquivos locais `.env` na VM. Dessa forma, as variáveis sensíveis do banco de dados nunca sobem para o repositório público do GitHub.
